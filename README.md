## 项目概述

### 业务背景

对于不通过零售通 POS 开单的门店，核销会员券时缺少 SN（设备序列号），无法进行合规性校验。本项目通过新增移动端 H5 页面，在核销环节增加 SN/IMEI 录入与实时校验，同时提供 PC 后台报表供总部查看核销数据。

### 核心链路

```
扫码/手动输入 SN/IMEI → 合规性校验（锁定/首销/机码上报/PO激活）
        ↓ 通过
扫码/手动输入会员券码 → 券码校验
        ↓ 通过
提交核销（自动上报机码 + 券码核销）
        ↓
PC后台实时查看核销历史数据
```

## 功能模块说明

### 移动端 H5（mobile-app.html）

#### 首页（工作台）
- 店铺信息卡片（店铺名称、经销商、面积、人员）
- 营销活动 Banner 轮播
- 金刚区导航（8 个入口，其中"门店会员营销"进入核销页）
- 销售看板（日/周/月维度数据指标）
- 底部 TabBar（工作台/库存管理/联想POS/个人中心/AI助手）

#### 核销页（券码核销）
- **步骤条**：3 步流程（SN/IMEI 校验 → 券码校验 → 提交核销）
- **SN/IMEI 录入**：
  - 扫一扫：调起通用扫码控件，1.5s 模拟扫码成功，自动回填并触发校验
  - 手动录入：弹窗输入编码，提交后触发校验
  - 校验场景：
    - 正常编码 → 校验通过
    - 包含 `fail` → 校验失败（通用错误）
    - 输入 `testerror` → 校验失败（设备不合规，展示完整合规性错误文案）
- **券类型选择**：页面顶部新增券类型下拉单选，支持 立减券/满减券/折扣券/实物券；切换券类型时自动重置表单
- **SN/IMEI 录入**：
  - 立减券/满减券/折扣券（默认）：显示 SN/IMEI 录入卡片，支持扫码/手动，校验通过后才允许录入会员券码
  - 实物券：隐藏 SN/IMEI 录入卡片，仅录入会员券码
- **券码录入**：仅支持一次核销一个会员券码，支持扫码/手动录入，单张独立校验通过/失败；需要 SN 的券类型下，SN/IMEI 校验未通过前券码区域禁用并提示"请先完成 SN/IMEI 校验，再录入会员券码"
- **提交核销**：所需录入项均校验通过后启用（实物券仅需券码通过，其他类型需 SN/IMEI 与券码均通过），提交后显示处理中 → 结果弹窗仅展示核销状态（成功/失败）、失败错误提示及确认按钮 → 成功关闭后自动重置

#### 核销记录页
- 搜索框（按核销券名称/商品名称模糊搜索）
- 筛选面板（10 个维度：券码、券名称、核销时间、核销金额、SN/IMEI、PN、商品分类、商品名称、核销状态、上报机码状态）
- 商品分类级联选择器（大类 → 中类 → 小类，三级联动）
- 记录卡片列表（含完整核销信息字段，新增【核销状态】【上报机码状态】；失败时分别支持【重新核销】【重新上报】）
- 全量导出 CSV

#### 扫码控件
- 全屏黑色遮罩 + 240×240 取景框（4 个白色角标）
- 蓝色扫描线动画（1.8s 循环）
- 关闭按钮（✕）、闪光灯按钮（⚡）
- 标题 + 提示文案（动态：扫描SN/IMEI 或 扫描券码）
- 2.5s 后模拟成功 → 绿色 ✓"扫码成功" → 0.8s 自动关闭返回

### PC 后台（pc.html）

#### 报表页面
- 左侧导航菜单：
  - `POS > 数据 > 会员券核销历史记录`
  - `非POS > 数据 > 会员券核销历史记录`（新增，页面内容与 POS 入口完全一致）
- 面包屑随菜单入口动态切换
- 顶部筛选面板（战区/门店/券名称/PN/经销商/店面类型/核销时间/商品分类）
- 数据表格（40 列），含分页、排序
- 字段取值样式规范：
  - **是/否字段**（7 个）：值为"否"时用正红色（`#FF4D4F`）浅底标签，"是"为纯黑文本
  - **其他字段**：统一纯黑文本
- CSV 导出

#### 报表字段清单（35 列）

| 分类 | 字段 |
|------|------|
| 基础信息 | 战区、K码、K渠道名称、D码、D门店名称、店面类型 |
| 券信息 | 核销券码ID、核销券码、券名称、券类型、活动名称、活动ID (TID)、会员手机号、联想会员ID、核销时间、核销金额 |
| 设备信息 | SN/IMEI、核销PN、核销店员 |
| 商品信息 | 商品分类、商品名称 |
| 合规校验 | 是否有效核销且PO合规、是否锁定、上报机码标识、上报机码状态 |
| 通路产品 | 所属通路、产品组 |
| PO校验 | PO是否合规、PO是否合规（地址校验） |
| 区域匹配 | 战区匹配、省份匹配、城市匹配 |
| SO数据 | so财年、so财季、so自然月、so日期、so战区、so省份、so城市、so区县 |

## 开发指南

### 本地运行

```bash
# 直接浏览器打开即可（无需构建）
open index.html    # macOS
start index.html   # Windows

# 或使用任意静态服务器
npx serve .
python -m http.server 8080
```

### 部署流程

项目使用 GitHub Pages 部署，`master` 分支为源码，`gh-pages` 分支为部署产物。

```bash
# 部署步骤
cp index.html pc.html mobile.html /path/to/deploy_dir/
cd /path/to/deploy_dir
git push -f origin master:gh-pages
```

### 从 Demo 到生产的改造要点

1. **工程化**：引入 Vite + TypeScript + ESLint + Prettier
2. **组件库**：移动端使用 Vant 4，PC 端使用 Element Plus
3. **路由**：使用 Vue Router 替换当前的 `v-if` 页面切换
4. **状态管理**：使用 Pinia 管理全局状态（店铺信息、核销状态等）
5. **网络层**：使用 Axios 封装 API 请求，统一错误处理
6. **扫码**：移动端接入企微 JSSDK `wx.scanQRCode`
7. **鉴权**：对接企微 OAuth 登录，获取用户身份和门店权限
8. **数据权限**：移动端按门店维度隔离数据，PC 端支持跨门店查询
9. **单元测试**：核心校验逻辑（SN 校验、券码校验、提交流程）编写单元测试
10. **埋点**：关键操作节点增加业务埋点（核销提交、校验失败等）
